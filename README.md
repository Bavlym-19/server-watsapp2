# سيرفر واتساب للربط مع  ender و Replit (متعدد الجلسات)
server 2
هذا المشروع يتيح لك إنشاء سيرفر واتساب يعمل على Render، حيث يمكنك إدارة ما يصل إلى **3 جلسات واتساب منفصلة** في نفس الوقت. لكل جلسة، يمكنك اختيار الربط إما عن طريق مسح QR Code أو باستخدام كود الربط (Pairing Code) الذي يظهر على الشاشة.

## 🚀 خطوات الرفع على Render

1.  قم بإنشاء مستودع (Repository) جديد على GitHub وارفع عليه هذه الملفات.
2.  ادخل على [Render.com](https://render.com) وقم بإنشاء **Web Service** جديدة.
3.  اربط حساب GitHub الخاص بك واختر المستودع الذي رفعته.
4.  في إعدادات Render:
    -   **Runtime**: `Docker` (سيتم التعرف عليه تلقائياً من ملف Dockerfile).
    -   **Plan**: يمكنك البدء بالخطة المجانية (Free Tier).
5.  بعد انتهاء الرفع، سيعطيك Render رابطاً مثل `https://your-app.onrender.com`.

## 📱 طريقة الاستخدام (إدارة الجلسات)

1.  افتح رابط السيرفر الذي حصلت عليه من Render في متصفحك.
2.  ستجد 3 أقسام للجلسات (الجلسة 1، الجلسة 2، الجلسة 3).
3.  لكل جلسة، يمكنك اختيار طريقة الربط:
    -   **بدء بالـ QR**: اضغط على هذا الزر وسيظهر لك QR Code. امسحه باستخدام واتساب من هاتفك (Linked Devices).
    -   **بدء بكود الربط**: اضغط على هذا الزر، ثم أدخل رقم هاتفك (مع مفتاح الدولة، مثال: 201234567890) واضغط "تأكيد كود الربط". سيظهر لك كود مكون من 8 أرقام. أدخل هذا الكود في واتساب على هاتفك (عبر خيار "ربط جهاز باستخدام رقم الهاتف" في إعدادات الأجهزة المرتبطة).
4.  بمجرد ظهور رسالة "تم الاتصال بنجاح"، تكون الجلسة جاهزة لاستقبال الأوامر.

## 🔗 الربط مع Replit (إرسال الرسائل)

داخل مشروعك في Replit، استخدم كود Python الموجود في ملف `replit_example.py`:
1.  انسخ الكود وضعه في ملف `main.py` في Replit.
2.  قم بتغيير قيمة `RENDER_SERVER_URL` إلى الرابط الخاص بك.
3.  عند استخدام الدالة `send_whatsapp_message`، ستحتاج الآن لتحديد `session_id` (مثال: `'session1'`, `'session2'`, `'session3'`) بالإضافة إلى رقم الهاتف والرسالة.

    ```python
    # مثال لإرسال رسالة للجلسة الأولى
    send_whatsapp_message('session1', '201234567890', 'مرحباً من الجلسة الأولى!')
    ```

## ⚠️ ملاحظة هامة
الخطة المجانية في Render تدخل في وضع "النوم" (Sleep) إذا لم يكن هناك نشاط. لإبقاء السيرفر مستيقظاً، يمكنك استخدام خدمة مثل [Cron-job.org](https://cron-job.org) لعمل "Ping" للرابط كل 10 دقائق.

### 🗑️ مسح بيانات الجلسة
إذا أردت تسجيل الخروج من جلسة معينة وإعادة ربطها، يمكنك حذف مجلد `auth_info_baileys_SESSIONID` (مثال: `auth_info_baileys_session1`) من ملفات مشروعك على Render (عبر واجهة Render أو يدوياً إذا كنت تستخدم Docker Volume) ثم إعادة تشغيل السيرفر. هذا سيجبر الجلسة على طلب QR Code أو كود ربط جديد.
